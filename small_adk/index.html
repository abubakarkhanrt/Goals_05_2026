Google ADK 2.1 Workshop lab

Riddle Agent (small_adk)

A small multi-file agent that teaches LlmAgent + FunctionTools, ADK callbacks (cloud→local fallback), and how that differs from a full Workflow pipeline in the parent repo.

1. What is an AI agent?

An AI agent uses a language model to decide what to do next — not just what to say. It can call tools (Python functions) to fetch data, update session state, and run multi-step reasoning across turns.

  • Chatbot — text in, text out, no side effects.
  • Agent — may call tools, keeps secrets in session state, branches on user input.

This lab plays a riddle game: load riddles, validate guesses, give hints, reveal answers only on give-up, and exit cleanly with /exit.

2. Google ADK 2.1 — Agents & Workflows

Google ADK (Agent Development Kit) version 2.1 offers two orchestration styles:

LlmAgent (Agent)

The LLM chooses when to call tools. Good for interactive, branching tasks.

This lab uses this pattern.

Workflow

A directed graph of nodes (gates, routers, tool nodes). Tools are invoked by the graph — not by LLM tool-calling.

See the main code/agent.py transcript verifier in the parent repo.

User message │ ▼ ┌─────────────────────┐ ┌────────────────────────────────────┐ │ LlmAgent │────▶│ tools.py (8 FunctionTools) │ │ (agent.py) │ │ start_riddle, check_answer, ... │ │ LLM + callbacks │◀────│ game.py holds secret answers │ └─────────────────────┘ └────────────────────────────────────┘ │ ▼ runtime.py — before_model / after_model / on_model_error (Gemini 503 → Ollama + offline riddles) vs. User message → Workflow gate → route → tool nodes → LLM assess (main project)

3. Code layout — read in this order

  1. riddles.json + constants.py — data and shared keys
  2. game.py — session state, guess/hint/exit logic, local-fallback handler
  3. tools.py — eight ADK FunctionTools the LLM calls
  4. runtime.py — LLM routing, connectivity, ADK callbacks
  5. agent.py — instruction + root_agent wiring (~100 lines)
riddles.json ──▶ riddles.py constants.py ──┐ ├──▶ game.py ◀── runtime.py (callbacks import game helpers) └──▶ tools.py ──▶ game.py agent.py ── imports tools.py + runtime.py ──▶ root_agent

4. Runtime modes

The agent picks its LLM and riddle source at startup (override with .env):

ConditionLLMRiddle source
Internet + GOOGLE_API_KEYGemini (cloud)Online API → LLM-generated fallback
Internet, no API keyOllama (local)Online API → LLM-generated fallback
Internet offOllama (local)8 offline riddles (riddles.json)
Gemini 503 / rate limitOllama (auto-fallback)Offline riddles via deterministic handler — resend your message
WORKSHOP_MODE=localOllama (forced)Offline riddles only — best for live demos

Extension pattern: ADK callbacks (runtime.py)

CallbackWhat it does
before_model_callbackOn fallback: short-circuit to game.py (no Ollama tool loop). Handles /exit instantly.
after_model_callbackRemap hallucinated tool names (e.g. riddle_master → start_riddle)
on_model_error_callbackGemini 503/429 → switch to Ollama, ask user to resend
on_tool_error_callbackRecover from invented tool names without crashing

Tools (tools.py)

ToolPurpose
start_riddlePreferred. Cached connectivity probe + load riddle in one LLM step
check_internetProbe connectivity (result cached 60s per session)
fetch_online_riddleHTTP API fetch; LLM generation on API failure
get_local_riddleRandom offline riddle from riddles.json
check_answerNormalized match; never returns the secret answer
give_hintPre-stored hints (offline) or LLM-generated (online)
reveal_answerOnly when the user explicitly gives up
exit_gameEnd the game — also triggered by /exit, quit, stop

Game flow

  1. Let's play or new riddle → start_riddle → question only (no answer)
  2. Wrong guess → “Are you sure? Is that your final answer?”
  3. Confirmed wrong → keep trying / hint — no auto-reveal
  4. give up → reveal_answer
  5. Correct → immediate congratulations
  6. /exit → leave game; say Let's play to start again
Design rule: The correct answer lives in ADK session state. Tools never leak it to the LLM until reveal_answer runs. On cloud fallback, game.py runs the game deterministically (no small-model tool loops).

5. Setup

Prerequisites

  • Python 3.11+ — required. System python3 on 3.8/3.10 only finds google-adk 0.0.1 and fails.
  • Ollama running (ollama serve) with your model pulled
  • Optional: GOOGLE_API_KEY for Gemini when online
Python check: python3.11 --version must work before creating the venv. Do not use bare python3 -m venv if it points to 3.8.

Install & run

cd small_adk
python3.11 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
python --version                  # must show 3.11.x
pip install --upgrade pip
pip install -r requirements.txt   # google-adk + litellm (~300 MB)
cp .env.example .env              # add GOOGLE_API_KEY if you have one

# From the parent of small_adk/ (folder that contains small_adk):
adk web
# → select small_adk

6. Workshop demo script (~10 min)

  1. Start: adk web → select small_adk
  2. Let's play — trace shows start_riddle (one tool call)
  3. Guess wrong → agent asks for final confirmation
  4. hint → pre-stored or generated hint
  5. give up → reveal_answer in trace
  6. /exit → game ends; Let's play starts fresh
  7. Fallback beat (optional): if Gemini 503s → resend message → offline riddle via Ollama path
  8. Offline beat: set WORKSHOP_MODE=local or disconnect Wi‑Fi → new riddle
