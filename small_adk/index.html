Concept

What is an AI agent?

Not just a chatbot — an agent decides what to do next and can call tools.

💬 Chatbot

Text in, text out. No side effects, no session secrets.

🤖 Agent

Calls tools, keeps secrets in session state, branches on user input.

This lab plays a riddle game: load riddles, validate guesses, give hints, reveal only on give-up, exit with /exit.

ADK 2.1

Agents vs Workflows

🧠 LlmAgent

Who decides? LLM picks tools per turn.

Best for: Open-ended chat — hint, guess, give up in any order.

This lab uses this.

🔀 Workflow

Who decides? Graph picks nodes per turn.

Best for: Fixed pipelines — same steps every time, easy to test.

See code/agent.py in the parent repo.

Combine both when you need a predictable tool chain and conversational play — e.g. parent repo: Workflow runs math → spatial → dates; LLM assesses at the end.

LLM orchestrates tools · Workflow orchestrates nodes

LLM & riddle source modes

Picked at startup from .env — auto-fallback on Gemini 429/503.

ConditionLLMRiddle source
Internet + GOOGLE_API_KEYGeminiOnline API → LLM fallback
Internet, no keyLocal deterministic handler8 offline riddles (avoids Ollama tool loops)
RAW_OLLAMA_MODE=true, no keyOllamaOne local-model call per turn for wording
Internet offLocal deterministic handler8 offline riddles
Gemini 429 / 503Ollama (auto)Offline via game.py — resend message
WORKSHOP_MODE=localOllama (forced)Offline only — best for demos
Local note: no-key/offline modes short-circuit in before_model by default. Set RAW_OLLAMA_MODE=true when you want to show the local model being called without ADK tool loops.
Extension

ADK callbacks (runtime.py)

CallbackWhat it does
before_modelFallback → short-circuit to game.py. Handles /exit instantly.
after_modelRemap hallucinated tool names → valid tools
on_model_errorGemini 429/503 → Ollama + real error message
on_tool_errorRecover from invented tool names

Callbacks intercept before the LLM runs

Tools

FunctionTools (tools.py)

ToolPurpose
start_riddlePreferred. Probe + fetch in one LLM step
check_internetConnectivity probe (cached 60s)
fetch_online_riddleHTTP API; LLM generation on failure
get_local_riddleRandom offline riddle
check_answerNormalized match — never leaks secret
give_hintPre-stored (offline) or LLM-generated
reveal_answerOnly on explicit give-up
exit_gameEnd game — also /exit, quit, stop
Design rule: Correct answer lives in session state. Tools never leak it until reveal_answer.

Setup

Install & run

  • Python 3.11+ — system 3.8/3.10 only finds google-adk 0.0.1
  • Ollama configured if you want to test raw local LLM calls
  • Optional: GOOGLE_API_KEY for Gemini when online
cd small_adk
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env

# From parent folder:
adk web   # → select small_adk
401 with API key? Set GOOGLE_GENAI_USE_VERTEXAI=FALSE.
WORKSHOP_MODE=local
# RAW_OLLAMA_MODE=true  # comment WORKSHOP_MODE first
SKIP_STARTUP_PROBE=true
OLLAMA_MODEL=ollama/your-model
OLLAMA_API_BASE=http://localhost:11434
GOOGLE_GENAI_USE_VERTEXAI=FALSE
